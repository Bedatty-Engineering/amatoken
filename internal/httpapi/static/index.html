Refresh & Auto-refresh

Refresh now
Triggers an immediate re-scan of ~/.claude/projects. The watcher already ingests new lines via fsnotify, but if a session was started while the container was off — or fsnotify dropped an event — this forces a full reconcile from each file's last byte offset. ✓ green after success, ✗ red on error (auto-clears in ~2.5s).
Auto-refresh
When on, the UI polls Refresh every 60s in the background. Same cadence as the server-side reconcile tick. Off by default — turn on if you keep the dashboard open during long Claude Code sessions and want totals to keep updating without manual clicks. Setting is persisted server-side.

What am I looking at?

One row per assistant message in this session — the granular events that Anthropic actually charges for. Synthetic events (context compaction, system prompts) are excluded.

Time
When the response was generated.
Model
Which model answered. Can vary mid-session if you used /model.
Branch
Git branch at that moment. May change if you switched branches mid-session.
Input
New tokens you sent in this request. Priced at base rate.
Output
Tokens Claude generated. Priced at ~5× input rate — usually the biggest cost driver.
Cache R/W
Cache read (cheap, ~10% of input rate) plus cache write (expensive, ~125% of input rate). Lots of cache writes = context kept changing.
Cost
Per-message cost in USD using the active pricing table. Sums up to the session total in the header.

Use this to spot which specific messages dominate the bill — a single 5k-output reply on Opus can cost $0.15+ alone. The raw conversation text isn't shown here; that's in the JSONL file on disk.
